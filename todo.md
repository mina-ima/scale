## 0. 事前準備 / 環境（P0）

- [ ] パッケージマネージャを選定して固定（`pnpm` 推奨）。
- [ ] Android/Chrome 最新端末を準備（HD/FHD/WQHD の少なくとも1機種）。
- [ ] iOS/Safari 参考端末（任意）を準備。
- [ ] Vercel アカウント作成/権限確認（プロジェクト作成可）。
- [ ] リポジトリ可視性/ライセンス/ブランチ保護ルールを決定。

- [ ] Vite + React + TypeScript で雛形生成（`pnpm create vite@latest`）。

- [ ] `src/` のディレクトリを仕様の構成に合わせて作成。
- [ ] Zustand / Three.js / @types/webxr を追加インストール。

- [ ] GitHub Actions で CI（lint / type-check / unit / e2e）を追加。
- [ ] Vercel にプロジェクト連携（main ブランチ → Production / PR → Preview）。

## 2. ベース実装（P0）

- [ ] ルーティング: トップ（モード選択）/計測/成長記録の 3 画面。

- [ ] 環境判定: `supportsWebXR()` 実装と分岐（AR/フォールバック）。
- [ ] UI コンポーネント: ボタン、トースト、ダイアログ、タブ。

## 3. core/camera（P0）

- [ ] `getUserMedia` 取得、権限拒否時のエラー分類（`CAMERA_DENIED`）。

## 5. core/fallback（P0）

## 6. core/reference（P0→P1）

- [ ] 単位切替（cm/m）。

- [ ] ファイル名生成: `hakattake_YYYY-MM-DD_[項目]_[数値+単位].jpg`。
- [ ] 端末ギャラリーへの保存（File System Access / download フォールバック）。
- [ ] 自動保存のトースト通知。

## 11. エラーハンドリング（P0）

- [ ] エラー型 `ErrorState` 実装（code/message）。
- [ ] カメラ拒否 → 再試行導線/設定案内。
- [ ] WebXR 非対応/失敗 → フォールバック自動切替。
- [ ] 平面未検出 → ガイド表示/露出ヒント。
- [ ] 保存失敗 → 画像品質/解像度を下げて再試行。
- [ ] 基準物検出失敗 → ヒント/再撮影導線。
- [ ] 例外は ErrorBoundary + 各 core で捕捉、ユーザーへはダイアログ/トースト。
