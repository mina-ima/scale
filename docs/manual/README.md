# はかったけ（scale）取扱説明書

> インストール不要、ブラウザだけで使える「ものの長さ・大きさを測る」アプリです。お子さんの成長記録にも使えます。

## このアプリでできること

- 家具や部屋など、身のまわりのものの大きさを画面上で測れます（最大10メートル程度を想定）。
- お子さんの成長記録として、身長・足のサイズ・手のサイズを測れます。体重は手入力できます。
- 成長記録モードでは、測定結果（数値・日付つきの合成画像）を端末に保存できます。
- すべての処理が端末内で完結し、写真やデータは外部サーバーに送られません（プライバシー重視）。

> 注記: 機能はプロジェクト資料（GEMINI.md）に基づきます。AR対応端末では WebXR、非対応端末では基準物（A4用紙・カード・硬貨など）を使う写真ベースの測定にフォールバックします。

## 想定ユーザー・利用シーン

- メジャーが手元にないときに、サッと大きさを測りたい方。
- お子さんの身長・手足のサイズを定期的に記録したい保護者の方。
- 写真やデータを外部に送りたくない、プライバシーを重視する方。

## 動作環境・前提

- スマートフォン／パソコンの最近のブラウザ（Chrome / Safari など）。
- AR測定には WebXR 対応端末・カメラが必要です。
- WebXR 非対応の端末では、写真の中に基準物（A4用紙・クレジットカード・硬貨など）を写して縮尺を補正する方式で測定します。

## セットアップ

ふだん使う方は、公開URL（後述）をブラウザで開くだけで利用できます。自分で動かす（開発する）場合は次の手順です（パッケージマネージャーは pnpm 推奨）。

1. リポジトリを取得します。
   ```bash
   git clone https://github.com/mina-ima/scale.git
   cd scale
   ```
2. 依存パッケージをインストールします。
   ```bash
   pnpm install
   ```
3. 開発サーバーを起動します。
   ```bash
   pnpm dev
   ```
   表示されたローカルURLをブラウザで開きます。

## 使い方（ステップbyステップ）

### 測定モード（保存しない）

1. アプリを開き、測定モードを選びます。
2. カメラを対象物に向けます。
3. 画面上で始点・終点を指定すると、長さがリアルタイムに表示されます（このモードでは保存されません）。

### 成長記録モード（保存する）

1. 成長記録モードを選びます。
2. 身長・足のサイズ・手のサイズなどを測定します。体重は手入力できます。
3. 測定値・日付つきの合成画像が、端末のギャラリーに自動保存されます。

### WebXR 非対応の端末の場合

1. 測りたいものと一緒に、基準物（A4用紙・カード・硬貨など）を写して撮影します。
2. アプリが基準物の大きさを使って縮尺を補正し、寸法を計算します。

## 主な機能

| 機能 | 説明 |
| --- | --- |
| 測定モード | 家具・部屋などの長さをリアルタイム表示（保存なし） |
| 成長記録モード | 身長・足・手のサイズを測定、体重は手入力、合成画像を保存 |
| AR測定 | WebXR 対応端末でAR測定 |
| 写真ベース測定 | 非対応端末で基準物を使い縮尺補正 |
| 端末内処理 | 写真・データを外部送信しない |

## よくある質問・トラブルシューティング

- **Q. AR測定ができません。**
  A. 端末が WebXR に対応していない可能性があります。その場合は、基準物（A4用紙・カード・硬貨）を一緒に写す写真ベースの測定をご利用ください。
- **Q. 測定値がずれます。**
  A. 写真ベースの測定では、基準物が平らに・はっきり写っているほど精度が上がります。明るい場所で撮影してください。
- **Q. データは外部に送られますか？**
  A. いいえ。すべて端末内で処理・保存され、外部サーバーには送信されません。
- **Q. 測定モードのデータが残りません。**
  A. 測定モードは保存しない仕様です。記録を残したい場合は成長記録モードを使ってください。

## 開発者向けメモ

- **技術スタック**: Vite + React + TypeScript。3D/AR= Three.js + WebXR。フォールバックの画像処理= OpenCV（`opencv-ts` / CDNの opencv.js）。状態管理= Zustand。ルーティング= React Router。テスト= Vitest（単体）+ Playwright（E2E）。
- **エントリ**: `index.html` → `src/main.tsx`。ソースは `src/`（`core` / `features` / `components` / `utils` などに分割）。
- **起動コマンド**:
  ```bash
  pnpm install
  pnpm dev           # 開発サーバー
  ```
- **ビルド/その他**:
  ```bash
  pnpm build         # 本番ビルド（dist/ に出力）
  pnpm test          # 単体テスト（vitest）
  pnpm test:e2e      # E2Eテスト（playwright）
  pnpm typecheck     # 型チェック
  ```
- **デプロイ先**: Vercel（静的サイト）。`.vercel` プロジェクト名は `scale`。`main` ブランチを本番、PR でプレビューを自動デプロイ（GEMINI.md記載）。
- **本番URL**: 未確認（Vercel連携あり。具体的な公開URLはVercel/リポジトリ設定を参照）。
- **リポジトリ**: https://github.com/mina-ima/scale
